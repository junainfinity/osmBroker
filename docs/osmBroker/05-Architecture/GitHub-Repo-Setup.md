# GitHub repository setup

## What the user asked

> upload all to new github repo called osmBroker

## What goes in

Everything under `/Users/arjun/Projects/osmBroker/` *except* generated/derived artefacts.

Included:
- `Package.swift`, `Package.resolved`
- `Sources/osmBrokerCore/` (broker logic)
- `Sources/osmBroker/` (SwiftUI app + Resources)
- `Tests/osmBrokerCoreTests/` (XCTest)
- `Scripts/make-app-bundle.sh`
- `docs/osmBroker/` (the Obsidian vault — every architecture note, dev log, test log, task ledger)
- `cli-agent-broker-handover.md` (original port reference)
- `osmBroker PRD.pdf` (spec)
- `index.html` (the HTML prototype that inspired the design)

Excluded via `.gitignore`:
- `.build/` (SwiftPM derived build products)
- `.DS_Store` (macOS metadata)
- `osmBroker.app/` (signed ad-hoc bundle; regenerated by `Scripts/make-app-bundle.sh`)
- IDE / editor metadata (`.swiftpm/`, `.vscode/`, `.idea/`)
- `*.xcodeproj` (none present, but future-proof)

## Creation flow

```sh
git init -b main
git add .gitignore
git add .
git commit -m "Initial commit — osmBroker v0.3 (Phase 1.5 + dark mode)"

# Auth check first; falls through if already logged in
gh auth status

# Create + push in one shot
gh repo create osmBroker --public --source=. --push --description "Native macOS app that detects local AI CLIs (Claude Code, Codex, etc.) and exposes them as OpenAI/Anthropic-compatible HTTP endpoints over the LAN."
```

## What's NOT in this push (and why)

- **Secrets**: the API key default `osm-local-dev` is in the source and that's fine — it's a placeholder, never used in production. The user changes it in the UI.
- **Build cache**: `.build/` is excluded; clones run `swift build` to regenerate.
- **App bundle**: `osmBroker.app/` is excluded; `Scripts/make-app-bundle.sh` rebuilds it deterministically.
- **macOS noise**: `.DS_Store` is excluded.
